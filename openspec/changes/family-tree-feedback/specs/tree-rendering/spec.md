## ADDED Requirements

### Requirement: 子・配偶者の決定的な並び順
システムは、子の並び順を「生年が判明している場合は生年昇順、不明な場合は名前順」で決定しなければならない(SHALL)。生年が判明している子は不明な子より前(左側)に配置しなければならない(SHALL)。配偶者が複数いる場合の並び順は、婚姻イベントの日付が判明しているものは日付昇順、不明なものは家系図データ内の登録順としなければならない(SHALL)。並び順は選択中の人物(`main_id`)や描画の起点によらず、同一の `TreeDocument` に対しては常に同じ結果でなければならない(MUST)。

#### Scenario: 生年が判明している子の並び順
- **GIVEN** 人物Aに子B(1990年生)・子C(1985年生)がいる
- **WHEN** 家系図を描画する
- **THEN** Cが左、Bが右の順で表示される

#### Scenario: 生年不明の子の並び順
- **GIVEN** 人物Aに生年不明の子D(名前「花子」)・子E(名前「太郎」)がいる
- **WHEN** 家系図を描画する
- **THEN** 名前の辞書順(太郎が花子より前)で表示され、名前順は生年が判明している兄弟姉妹より後ろに配置される

#### Scenario: 性別が未設定でも並び順が変化しない
- **GIVEN** 性別が「不明」に設定された人物を含む家系図が表示されている
- **WHEN** 別の人物カードを選択し、描画の起点(`main_id`)が変わる
- **THEN** 子・配偶者の並び順は選択前と変わらない

### Requirement: 性別の視覚的表現
システムは人物カード上で性別(男・女・不明)を、色のみに依存しないアイコン・形状による視覚的インジケーターで表現しなければならない(SHALL)。このインジケーターは選択状態を示す朱のマーカーと異なる表現を用い、両者が混同されてはならない(MUST NOT)。性別が「不明」の人物は、性別を推測しない中立的な表現としなければならない(SHALL)。

#### Scenario: 性別が視覚的に判別できる
- **GIVEN** 性別「男」の人物Aと性別「女」の人物Bを含む家系図
- **WHEN** 家系図を描画する
- **THEN** AとBのカードは色に加えて形状・アイコンでも判別可能な性別インジケーターを表示する

#### Scenario: 性別不明の中立表現
- **GIVEN** 性別が未入力(「不明」)の人物C
- **WHEN** 家系図を描画する
- **THEN** Cのカードには男女いずれかを示唆しない中立的なインジケーターが表示される

### Requirement: 現年齢・没年齢の表示
システムは、生年月日が年月日まで判明している人物について、現在日時時点の満年齢(死亡している場合は没年齢)を人物カードに補助情報として表示しなければならない(SHALL)。生年が年のみしか判明していない場合は、誤解を招く概算表示を避けるため年齢を表示してはならない(MUST NOT)。

#### Scenario: 存命人物の現年齢表示
- **GIVEN** 生年月日が「1990年5月1日」で死亡記録のない人物
- **WHEN** 家系図を描画する
- **THEN** 人物カードに現在日時時点の満年齢が表示される

#### Scenario: 故人の没年齢表示
- **GIVEN** 生年月日「1900年1月1日」・没年月日「1980年1月1日」の人物
- **WHEN** 家系図を描画する
- **THEN** 人物カードには没年齢(80歳)が表示され、現在日時からの年齢は表示されない

#### Scenario: 年のみ判明している場合は年齢を表示しない
- **GIVEN** 生年が「1900年頃」(年のみ判明)の人物
- **WHEN** 家系図を描画する
- **THEN** 人物カードに年齢は表示されない

### Requirement: 折りたたみ表示時の非表示人数バッジ
システムは、折りたたみ表示(全体表示モードを使用していない状態)において、現在描画されていない人物へつながる境界の人物カードに、その先に存在する非表示人物の人数をバッジとして表示しなければならない(SHALL)。全体表示モードが有効な間はこのバッジを表示してはならない(MUST NOT)。

#### Scenario: 非表示の家族分岐がある場合のバッジ表示
- **GIVEN** 選択中の人物の視点からは描画されない配偶者の別の婚姻家族(子3名を含む)が存在する
- **WHEN** 折りたたみ表示で家系図を描画する
- **THEN** 当該配偶者のカードに非表示人数「3」を示すバッジが表示される

#### Scenario: 全体表示モードではバッジが表示されない
- **GIVEN** 折りたたみ表示で非表示人数バッジが表示されている状態
- **WHEN** 全体表示モードを有効にする
- **THEN** 該当していた人物のバッジは表示されなくなる

### Requirement: 全体表示モード
システムは、折りたたみ(単一視点での祖先/子孫展開)を行わず、`TreeDocument` 内で描画可能な最大範囲の人物を常に表示する「全体表示モード」を提供しなければならない(SHALL)。全体表示モードが有効な間、人物カードの選択操作は表示範囲(視点)を変更してはならない(MUST NOT)。全体表示モードと折りたたみ表示(デフォルト)は利用者がいつでも切り替えられなければならない(SHALL)。

#### Scenario: 全体表示モードの有効化
- **GIVEN** 折りたたみ表示で一部の傍系親族が非表示になっている家系図
- **WHEN** 全体表示モードを有効にする
- **THEN** 描画可能な範囲の人物が漏れなく表示される

#### Scenario: 全体表示モード中は選択しても視点が動かない
- **GIVEN** 全体表示モードが有効な状態
- **WHEN** 表示範囲の端に近い人物カードを選択する
- **THEN** 選択状態(朱のマーカー)のみが変わり、表示されている人物の範囲は変化しない

## MODIFIED Requirements

### Requirement: 複雑な家族関係の描画
システムは再婚・複数配偶者を同一図上に描画できなければならない(SHALL)。養子の系線は実子と視覚的に区別できる表現(破線等)で描画しなければならない(MUST)。同一人物が複数の家族に「子」として属する場合(実親と養親の両方に記録されている等)、システムは実子(`biological`)以外の続柄(養子・継子・里子・不明)を優先して主たる親子線として描画しなければならない(SHALL)。複数の婚姻関係を持つ人物については、婚姻イベントの日付が判明している婚姻を日付昇順に並べて描画しなければならない(SHALL)。

#### Scenario: 再婚の描画
- **GIVEN** 人物Aに配偶者B(離別)と配偶者Cの2つのFamilyがある
- **WHEN** 家系図を描画する
- **THEN** Aから2本の婚姻線がB・Cへ描かれ、それぞれの子が正しい親の下に配置される

#### Scenario: 養子の系線
- **GIVEN** 人物Dが養親のFamilyに続柄「養子」で帰属している
- **WHEN** 家系図を描画する
- **THEN** Dへの系線は実子の系線と異なるスタイル(破線)で描かれ、凡例等でその意味が分かる

#### Scenario: 実親・養親の両方が記録された人物の描画
- **GIVEN** 人物Eが実親のFamily(続柄`biological`)と養親のFamily(続柄`adopted`)の両方に子として記録されている
- **WHEN** 家系図を描画する
- **THEN** Eの主たる親子線は養親側(破線)として描画され、養子縁組の関係が家系図上から欠落しない

#### Scenario: 複数婚の時系列順表示
- **GIVEN** 人物Fに1858年に婚姻したG、1883年に婚姻したHの2つのFamilyがある
- **WHEN** 家系図を描画する
- **THEN** Gの婚姻線がHの婚姻線より先(時系列順)に配置され、どちらが先の婚姻かが左右の並びから読み取れる
