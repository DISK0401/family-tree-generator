# tree-rendering

family-chartを用いた家系図の描画・自動レイアウト・キャンバス操作。ドメインモデルを唯一の真実とし、変換アダプタ経由で描画する。独自デザイン言語(縦書き人名カード、意味を持つ系線)を実装する。

## ADDED Requirements

### Requirement: 自動レイアウト
システムは人物・関係の追加・変更・削除のたびに家系図を自動整列しなければならない(SHALL)。利用者による線の引き直しや位置の手動調整を必要としてはならない(MUST NOT)。

#### Scenario: 人物追加時の自動整列
- **GIVEN** 3世代・10人の家系図が表示されている
- **WHEN** 中間世代の人物に子を追加する
- **THEN** 線の交差や重なりを利用者が修正することなく、図全体が自動で再整列される

### Requirement: 描画アダプタによるモデル変換
描画はドメインモデル(TreeDocument)から描画ライブラリ形式への一方向変換アダプタを介して行わなければならない(SHALL)。描画ライブラリのデータ構造をアプリの保存・編集の正本として使用してはならない(MUST NOT)。

#### Scenario: モデル変更の描画反映
- **WHEN** ドメインモデルに人物追加コマンドを適用する
- **THEN** アダプタが再変換した結果のみに基づいて図が更新される(描画側で保持する独自状態に依存しない)

### Requirement: 複雑な家族関係の描画
システムは再婚・複数配偶者を同一図上に描画できなければならない(SHALL)。養子の系線は実子と視覚的に区別できる表現(破線等)で描画しなければならない(MUST)。

#### Scenario: 再婚の描画
- **GIVEN** 人物Aに配偶者B(離別)と配偶者Cの2つのFamilyがある
- **WHEN** 家系図を描画する
- **THEN** Aから2本の婚姻線がB・Cへ描かれ、それぞれの子が正しい親の下に配置される

#### Scenario: 養子の系線
- **GIVEN** 人物Dが養親のFamilyに続柄「養子」で帰属している
- **WHEN** 家系図を描画する
- **THEN** Dへの系線は実子の系線と異なるスタイル(破線)で描かれ、凡例等でその意味が分かる

### Requirement: キャンバス操作
システムはキャンバスのズーム・パン・人物カードの選択を提供しなければならない(SHALL)。選択中の人物は視覚的に強調(朱のマーカー)されなければならない(MUST)。

#### Scenario: ズームとパン
- **WHEN** ホイール操作(またはピンチ)とドラッグを行う
- **THEN** 図が拡大縮小・移動し、人物カードの文字は拡大時も鮮明に表示される

#### Scenario: 人物の選択
- **WHEN** 人物カードをクリックする
- **THEN** カードが朱色のマーカーで強調され、コンテキストアクションと編集パネルが有効になる

### Requirement: 縦書き人名カードとデザイントークン
人物カードの氏名は縦書き(`writing-mode: vertical-rl`)・明朝系書体で表示しなければならない(SHALL)。画面の配色・書体・余白はデザイントークン(墨・楮紙・藍・朱の4役+セマンティック色、ライト/ダーク両テーマ)として一元定義し、コンポーネントはトークン経由でのみ参照しなければならない(MUST)。

#### Scenario: 縦書き表示
- **WHEN** 氏名「山田 太郎」の人物カードを描画する
- **THEN** 氏名が縦書き・明朝系で表示され、生没年は補助情報として読める大きさで併記される

#### Scenario: ダークテーマ
- **WHEN** OSまたはアプリ設定でダークテーマが有効になる
- **THEN** トークンの再定義のみで全体が「藍鉄の地に生成りの文字」の配色に切り替わり、コントラストが読書可能な水準に保たれる

### Requirement: モーションの配慮
レイアウト変化時のアニメーションは、OSの`prefers-reduced-motion`が有効な場合に無効化されなければならない(SHALL)。

#### Scenario: モーション低減設定の尊重
- **GIVEN** OSで「視差効果を減らす」が有効
- **WHEN** 人物を追加して図が再整列する
- **THEN** トランジションなしで即座に最終配置が表示される
