## ADDED Requirements

### Requirement: 家族(婚姻・離婚)イベントの編集
システムは、選択中の人物が配偶者として属する家族(Family)ごとに、婚姻日・離婚日(各+場所)を編集できなければならない(SHALL)。編集は人物編集パネル内の独立した編集面として提供し、確定操作を必要とせず変更を即座にデータモデルへ反映しなければならない(SHALL)。日付入力は「和暦対応の日付入力」要件と同じ和暦・西暦相互入力を受け付けなければならない(MUST)。

#### Scenario: 婚姻日の設定
- **GIVEN** 人物Aと人物Bの家族(Family)が存在する
- **WHEN** Aの編集パネルで当該家族の婚姻日欄に「昭和50年4月1日」と入力する
- **THEN** 家族の婚姻イベントとして日付が保存され、近傍に西暦表記「1975年4月1日」が即時表示される

#### Scenario: 離婚日の設定
- **GIVEN** 婚姻日が記録済みのA-Bの家族が存在する
- **WHEN** 離婚日欄に日付を入力する
- **THEN** 家族の離婚イベントとして日付が保存される

#### Scenario: 即時反映(確定操作不要)
- **GIVEN** 家族イベント編集欄が表示されている
- **WHEN** 婚姻日を入力してフォーカスを外す
- **THEN** 別途「確定」操作を行わなくても直ちにデータモデルへ反映される(人物編集パネルの離脱確認ダイアログは発生しない)

#### Scenario: 複数配偶者を持つ人物の編集
- **GIVEN** 人物Aに配偶者B(離婚済み)と配偶者C(現配偶者)の2つの家族がある
- **WHEN** Aの編集パネルを開く
- **THEN** A-B、A-Cそれぞれの家族について独立した婚姻・離婚日欄が表示され、一方の変更が他方に影響しない

#### Scenario: 日付の削除
- **GIVEN** 婚姻日が設定されている家族がある
- **WHEN** 婚姻日欄を空にする
- **THEN** その家族の婚姻イベントが削除され、他のイベント(離婚日等)は影響を受けない

#### Scenario: 3件以上のイベントを持つ家族の編集
- **GIVEN** GEDCOMインポート等により、婚姻→離婚→婚姻(復縁)の3件のイベントを持つ家族が存在する
- **WHEN** その家族の編集欄を開く
- **THEN** 種別(婚姻/離婚)ごとに最初の1件のみが編集対象として表示され、それ以外のイベントが存在する旨の注記が表示される。この状態で他のフィールドを編集・保存しても、編集対象外のイベントはデータから失われない
