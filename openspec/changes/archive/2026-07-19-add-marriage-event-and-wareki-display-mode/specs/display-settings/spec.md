## ADDED Requirements

### Requirement: 和暦表示モード
システムは、人物カードに表示する生年月日・没年月日の表記形式として西暦・和暦を選択できなければならない(SHALL)。和暦表示を選択した場合、既存の日付粒度設定(年のみ/年月まで/年月日まで)と組み合わせて丸めた上で和暦表記に変換して表示しなければならない(SHALL)。この設定は表示専用であり、`TreeDocument`(GEDCOM/JSONエクスポート内容)には影響してはならない(MUST NOT)。

#### Scenario: 和暦表示への切り替え
- **GIVEN** 生年月日「1964年10月10日」を持つ人物が家系図に含まれる
- **WHEN** 表示設定で表示形式を「和暦」に変更する
- **THEN** 人物カードの生年月日表示が「昭和39年10月10日」に変わる

#### Scenario: 粒度設定との組み合わせ
- **GIVEN** 表示形式が「和暦」、生年月日の粒度が「年のみ」に設定されている
- **WHEN** 生年月日「1964年10月10日」を持つ人物のカードを表示する
- **THEN** 「昭和39年」とだけ表示され、月日は表示されない

#### Scenario: 元号テーブル範囲外の日付
- **GIVEN** 明治より前(1868年より前)の生年月日を持つ人物が家系図に含まれる
- **WHEN** 表示形式が「和暦」に設定されている
- **THEN** 当該人物のカードは西暦表記のまま表示される(和暦への変換対象外として扱われる)

#### Scenario: エクスポートに影響しない
- **GIVEN** 表示形式で「和暦」を選択している
- **WHEN** GEDCOMまたはJSON形式でエクスポートする
- **THEN** エクスポートされたファイルには西暦の日付情報がそのまま出力される

#### Scenario: 設定の永続化
- **GIVEN** 表示形式を「和暦」に変更した
- **WHEN** ページを再読み込みする
- **THEN** 表示形式は「和暦」のまま復元される

### Requirement: カード表示項目の選択
システムは、人物カードに表示する項目(姓・名・ふりがな・生年月日・没年月日・出生地・没地・年齢・性別アイコン)を、利用者が項目ごとに独立して表示・非表示を選択できなければならない(SHALL)。各項目の表示・非表示はすべての人物カードに一律に適用されなければならない(SHALL)。この設定は表示専用であり、`TreeDocument`には影響してはならない(MUST NOT)。

#### Scenario: 姓を非表示にして名だけ表示する
- **GIVEN** 姓「山田」・名「太郎」を持つ人物が家系図に含まれる
- **WHEN** 表示設定で「姓」の表示をオフにする
- **THEN** 人物カードには「太郎」のみが表示され、「山田」は表示されない

#### Scenario: ふりがなを表示する
- **GIVEN** ふりがな「やまだ」「たろう」が設定された人物が家系図に含まれる
- **WHEN** 表示設定で「ふりがな」の表示をオンにする
- **THEN** 人物カードに氏名とあわせてふりがなが表示される

#### Scenario: 出生地・没地を表示する
- **GIVEN** 出生地「東京都」が記録された人物が家系図に含まれる
- **WHEN** 表示設定で「出生地」の表示をオンにする
- **THEN** 人物カードに出生地が表示される

#### Scenario: 姓・名を両方非表示にする
- **GIVEN** 姓・名が設定された人物が家系図に含まれる
- **WHEN** 表示設定で「姓」「名」の両方の表示をオフにする
- **THEN** 人物カードの氏名欄は空欄になり、「名前未設定」等のフォールバック文言は表示されない

#### Scenario: 未設定時のデフォルト表示
- **GIVEN** 表示設定を一度も変更したことがない端末でアクセスする
- **WHEN** 家系図エディタを開く
- **THEN** 姓・名・生年月日・没年月日・年齢・性別アイコンが表示され、ふりがな・出生地・没地は表示されない(既存ユーザーが変更を意識することなく従来どおりの表示になる)

#### Scenario: 設定の永続化
- **GIVEN** 表示設定で「ふりがな」の表示をオンにした
- **WHEN** ページを再読み込みする
- **THEN** ふりがなの表示設定は維持される

### Requirement: 婚姻線ラベルの表示
システムは、配偶者を結ぶ婚姻線に婚姻日を表示するかどうかを利用者が選択できなければならない(SHALL)。オンの場合、家族の最初の婚姻イベントの日付を、和暦表示モードの設定に追従して線上に表示しなければならない(SHALL)。離婚日はこのラベルに含めてはならない(MUST NOT)。未設定時のデフォルトはオフでなければならない(SHALL)。この設定は表示専用であり、`TreeDocument`には影響してはならない(MUST NOT)。

#### Scenario: 婚姻線に婚姻日を表示する
- **GIVEN** 婚姻日「1990年4月1日」が記録された配偶者2人の家族が家系図に含まれる
- **WHEN** 表示設定で「婚姻日(線)」の表示をオンにする
- **THEN** 2人を結ぶ婚姻線上に「1990年4月1日」が表示される

#### Scenario: デフォルトでは非表示
- **GIVEN** 表示設定を一度も変更したことがない端末でアクセスする
- **WHEN** 婚姻日が記録された配偶者2人の家族を含む家系図を開く
- **THEN** 婚姻線に婚姻日は表示されない(既存ユーザーが変更を意識することなく従来どおりの表示になる)

#### Scenario: 和暦表示モードに追従する
- **GIVEN** 「婚姻日(線)」の表示がオン、表示形式が「和暦」に設定されている
- **WHEN** 婚姻日「1990年4月1日」が記録された家族の婚姻線を表示する
- **THEN** 婚姻線に「平成2年4月1日」が表示される

#### Scenario: 離婚済みでも婚姻線ラベルは表示され続ける
- **GIVEN** 婚姻日・離婚日の両方が記録された家族があり、「婚姻日(線)」の表示がオンになっている
- **WHEN** 家系図キャンバスを表示する
- **THEN** 婚姻線には婚姻日のみが表示され、離婚日は表示されない

#### Scenario: 設定の永続化
- **GIVEN** 表示設定で「婚姻日(線)」の表示をオンにした
- **WHEN** ページを再読み込みする
- **THEN** 「婚姻日(線)」の表示設定は維持される
