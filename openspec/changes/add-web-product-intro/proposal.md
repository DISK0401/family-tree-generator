# Proposal: Webトップページに製品紹介(ランディングページ)を用意する

## Why

現在、Webのトップページ(`/`)を開くと家系図エディタが直接表示され、初めて訪れたユーザーが「これが何のサービスか・何ができるか・データがどう扱われるか」を知る手段がない。競合調査(docs/research/2026-07-market-ux-legal.md)でも「使い方のイントロダクションがない」ことが国内最大手への代表的な不満として挙がっており、また新興のxGrapherは「インストール不要・モダンUI」を訴求して支持を得ている。BtoCサービスとして、画像・ビジュアルを用いたモダンな製品紹介ページを用意し、本サービスの差別化軸(プライバシーファースト・和暦対応・GEDCOM対応)を最初の数秒で伝えることで、利用開始率と信頼感を高める。

あわせて、今後実装予定の有償機能(戸籍スキャンからの自動生成、クラウド保存・親族共有など)を「Coming Soon」として掲載することで、ロードマップへの期待を醸成しつつ、現時点で提供している機能との線引きを明確にする。

さらに、完成した家系図のイメージを言葉ではなく実物で伝えるため、著作権・個人情報の問題がない偉人(故人であり、Wikipedia等で系譜が公知の人物)の家系図サンプルを複数パターン掲載し、そのままエディタで開いて触れられるようにする。「使ってみないと分からない」というオンボーディングの壁を、サンプル体験で最初に取り除く。

## What Changes

- トップページ(`/`)を製品紹介(ランディングページ)に変更し、家系図エディタは別パス(例: `/app`)へ移動する
  - **BREAKING(URL変更)**: 既存ユーザーがブックマークしている `/` はエディタではなくなる。ただしデータはIndexedDBに保持されたままであり、エディタの新URLへ誘導する導線を必ず用意する
- ランディングページに以下のセクションを設ける
  - ヒーロー: キャッチコピー+家系図のビジュアル(スクリーンショットまたはイラスト)+「無料ではじめる」CTA
  - 提供中機能の紹介: ブラウザ完結・サーバ送信なし(プライバシーファースト)、和暦⇄西暦対応の入力、自動整列される家系図描画、IndexedDB自動保存、GEDCOM/JSONインポート・エクスポート
  - Coming Soon(実装予定)の紹介: 戸籍スキャンからの自動生成(有償予定)、クラウド保存・パスワード付き親族共有(有償予定)、高品質な印刷/PDF出力 など。「Coming Soon」バッジで提供中機能と視覚的に区別する
  - 偉人家系図のサンプルギャラリー: 複数パターンのサンプルを図版で切り替え表示し、各サンプルを「エディタで開く」導線を設ける
    - 徳川家康(側室・多数の子女 = 複数配偶者パターン)、夏目漱石(養子縁組パターン)、渋沢栄一(再婚パターン)の3偉人 + 架空の現代家族(サービスの主用途を示すパターン。架空であることを明記)
    - すべて故人かつWikipedia等で系譜が公知の人物に限定する。実在の存命人物の系譜は掲載しない。皇室系譜は扱わない(法的には故人のみなら可能だが、商用サービスの宣伝への利用は批判リスクがあるため)
  - プライバシーへの姿勢: 無料版は家系図データ・個人情報を一切サーバへ送信しないことの明示
  - フッター: 注意書き(本サービスの家系図は観賞用・記念用であること)等
- エディタ側にサンプル読み込み経路を追加する: `/app?sample=<id>` で該当サンプルをエディタに展開する。既存の家系図データがある場合は、既存インポート機能と同様に上書き前の確認を必須とする(ユーザーデータの黙殺的な消失を防ぐ)
- BtoC向けにモダンなビジュアルデザインを採用する(画像・イラストの活用、レスポンシブ対応、ダーク/ライトの配色検討はdesignで決定)
- README.mdをページ構成の変更に合わせて更新する

## Capabilities

### New Capabilities
- `product-intro-page`: トップページとしての製品紹介(ランディング)ページ。ヒーロー・提供中機能・Coming Soon機能・プライバシー訴求・エディタへのCTA導線・レスポンシブ表示をカバーする
- `sample-tree-gallery`: 偉人家系図サンプルのギャラリー表示(ランディング側)と、サンプルをエディタへ展開する読み込み経路(`/app?sample=<id>`、既存データ保護の確認を含む)

### Modified Capabilities
- なし(エディタ・データモデル等の既存要件は変更しない。サンプル読み込みは既存のインポート系機構を再利用する新規経路であり、`tree-editor`・`json-import-export` の既存要件には手を入れない。エディタの配置URLは実装詳細としてdesignで扱う)

## Impact

- **対象**: 無料版のみ(有償版はComing Soon表記としての「紹介」のみで、機能実装は含まない)
- **コード**: `src/App.tsx` を起点とした画面構成(ランディングとエディタの出し分けのためのルーティング導入)、新規ランディング用コンポーネント・スタイル、`index.html` のtitle/meta(OGP等)の整備、`public/` への画像アセット追加、サンプル家系図データ(内部ドキュメント形式)の同梱とエディタ側の読み込み処理(既存の `tree-store.replace` と確認ダイアログのパターンを再利用)
- **依存関係**: ルーティング手段(軽量ルータ導入 or パス判定)はdesignで決定。大きな新規依存は想定しない
- **デプロイ**: Cloudflare Workers static assetsのSPA配信設定がサブパス(`/app`)直アクセスで404にならないか確認が必要
- **プライバシー・法務への影響**: 家系図データの流れは一切変わらない(ランディングは静的コンテンツで個人データを扱わない)。ただし紹介文言には以下を遵守する
  - 家系図は「観賞用・記念用」であることを明記し、相続関係説明図等の事実証明文書を作成できる印象を与えない
  - Coming Soonの戸籍スキャン機能は「利用者が自ら取得した戸籍を扱う」前提で記述し、取得代行を想起させない。精度の過度な約束(全自動生成等)をしない
  - サンプルの偉人は故人に限定する(死者の情報は個人情報保護法の対象外。系譜という事実情報に著作権は生じない)。系譜情報はWikipedia等で公知のものだけを使い、存命の子孫に言及しない。実在の存命人物・皇室系譜は扱わない
- **競合UXとの比較**: 最大手「みんなの家系図」はイントロダクション不足が不満点であり、製品紹介ページ+エディタ内の既存EmptyStateGuideの組み合わせで上回る。xGrapherの「インストール不要・モダンUI」訴求と同等以上の第一印象を目指す
- **ロールバック方針**: ランディングは既存エディタから独立した追加コンポーネントであり、ルーティングを戻せば従来どおり `/` でエディタが表示される状態に復帰できる。IndexedDBのデータ・スキーマには一切触れないためデータ面のロールバックは不要
