# gedcom-import-export

GEDCOM 7.0 / 5.5.1 のインポートと、GEDCOM 7.0 / 5.5.1互換モードのエクスポート。文字コードの取り扱い、非対応タグの保全、インポート結果サマリの提示を含む。

## ADDED Requirements

### Requirement: クライアント完結の処理(プライバシー)
GEDCOMのインポート・エクスポート処理は、すべてユーザーのブラウザ内で完結しなければならない(MUST)。処理の過程で家系図データ・ファイル内容・ファイル名を含むいかなる情報もサーバ・外部APIへ送信してはならない(MUST NOT)。

#### Scenario: インポート/エクスポート時にネットワーク送信が発生しない
- **WHEN** GEDCOMファイルのインポートおよびエクスポートを実行し、ブラウザ開発者ツールのNetworkタブを確認する
- **THEN** 自ホストの静的アセット取得以外のネットワーク通信が発生しない

### Requirement: GEDCOM 7.0エクスポート
システムは内部データモデルをGEDCOM 7.0形式のファイルとしてエクスポートできなければならない(SHALL)。出力はUTF-8(BOM付き)とし、ヘッダに `GEDC.VERS 7.0` を含まなければならない(SHALL)。人物・家族関係(続柄種別を含む)・日付・氏名(読み仮名を含む)をGEDCOM 7.0の対応構造にマッピングしなければならない(SHALL)。

#### Scenario: 基本的なエクスポート
- **GIVEN** 人物2名とその婚姻家族を含むデータモデル
- **WHEN** GEDCOM 7.0形式でエクスポートする
- **THEN** INDIレコード2件・FAMレコード1件を含むGEDCOM 7.0ファイルがダウンロードされ、ヘッダにバージョン7.0とUTF-8指定が含まれる

#### Scenario: 養子縁組のエクスポート
- **GIVEN** 実親の家族に「実子」、養親の家族に「養子」として属する人物を含むデータモデル
- **WHEN** GEDCOM 7.0形式でエクスポートする
- **THEN** 当該人物のINDIレコードに両家族へのFAMCリンクが出力され、養親側のリンクには続柄(PEDI adopted相当)が付与される

#### Scenario: 読み仮名のエクスポート
- **GIVEN** 姓「齋藤」読み仮名「さいとう」の人物を含むデータモデル
- **WHEN** GEDCOM 7.0形式でエクスポートする
- **THEN** NAMEには旧字体「齋藤」がそのまま出力され、読み仮名はTRAN(音訳)構造で出力される

#### Scenario: 和暦日付のエクスポート
- **GIVEN** 生年月日「明治十年頃」(西暦変換可能)の人物を含むデータモデル
- **WHEN** GEDCOM 7.0形式でエクスポートする
- **THEN** DATE値には西暦換算の日付(ABT修飾付き)が出力され、和暦の原文はPHRASE構造で保全される

### Requirement: GEDCOM 5.5.1互換モードエクスポート
システムはGEDCOM 5.5.1互換形式でのエクスポートを選択できなければならない(SHALL)。出力はUTF-8とし、ヘッダに `VERS 5.5.1` と `CHAR UTF-8` を含まなければならない(SHALL)。5.5.1で表現できない情報(7.0固有の構造)は欠落させず、NOTEまたは慣行タグで保全し、変換されなかった項目を警告として提示しなければならない(SHALL)。

#### Scenario: 5.5.1互換モードでのエクスポート
- **GIVEN** 人物と家族を含むデータモデル
- **WHEN** GEDCOM 5.5.1互換モードでエクスポートする
- **THEN** ヘッダに5.5.1とCHAR UTF-8が出力され、他サービス(5.5.1対応ソフト)が読み込める構造のファイルが生成される

#### Scenario: 5.5.1で表現できない情報の警告
- **GIVEN** 事実婚など5.5.1に直接の対応構造がない関係種別を含むデータモデル
- **WHEN** GEDCOM 5.5.1互換モードでエクスポートする
- **THEN** 当該情報はNOTE等で保全され、「5.5.1では標準表現できないため代替表現で出力した」旨の警告が表示される

### Requirement: GEDCOMインポートとバージョン自動判定
システムはGEDCOM 7.0および5.5.1のファイルをインポートし、内部データモデルへ変換できなければならない(SHALL)。バージョンはファイルヘッダから自動判定しなければならない(SHALL)。ヘッダが欠落・不正でGEDCOMとして解釈できない場合はインポートを中断し、理由を平易な日本語で表示しなければならない(SHALL)。

#### Scenario: GEDCOM 7.0ファイルのインポート
- **WHEN** GEDCOM 7.0形式のファイルを選択またはドラッグ&ドロップでインポートする
- **THEN** バージョンが自動判定され、人物・家族が内部データモデルに変換される

#### Scenario: GEDCOM 5.5.1ファイルのインポート
- **WHEN** 他サービスがエクスポートしたGEDCOM 5.5.1ファイルをインポートする
- **THEN** バージョンが自動判定され、人物・家族・読み仮名タグ(_KANA/ROMN)が内部データモデルに変換される

#### Scenario: GEDCOMとして解釈できないファイル
- **WHEN** GEDCOMヘッダを持たないテキストファイルをインポートしようとする
- **THEN** インポートは中断され、「GEDCOMファイルとして読み込めない」旨が平易な日本語で表示され、既存のデータは変更されない

#### Scenario: 養子縁組を含むファイルのインポート
- **GIVEN** PEDI adoptedを含むGEDCOMファイル
- **WHEN** インポートする
- **THEN** 当該の子リンクは続柄種別「養子」として内部データモデルに変換される

### Requirement: 文字コードの自動判定
インポート時、システムはUTF-8(BOM有無とも)・UTF-16(BOM付き)・Shift_JISのファイルを自動判定して読み込めなければならない(SHALL)。判定はUTF-8厳格デコードを最優先し、失敗時のみShift_JISへフォールバックしなければならない(SHALL)。判定したエンコーディングをインポート結果サマリに表示しなければならない(SHALL)。ANSELエンコーディングを検出した場合はインポートを中断し、理由を表示しなければならない(SHALL)。

#### Scenario: UTF-8ファイルのインポート
- **WHEN** UTF-8(BOMなし)のGEDCOMファイルをインポートする
- **THEN** 文字化けなく読み込まれ、サマリに「UTF-8として読み込んだ」旨が表示される

#### Scenario: Shift_JISファイルのインポート
- **GIVEN** 国産ソフトが出力したShift_JISエンコーディングのGEDCOMファイル
- **WHEN** インポートする
- **THEN** 日本語(氏名・地名)が文字化けなく読み込まれ、サマリに「Shift_JISとして読み込んだ」旨が表示される

#### Scenario: ANSELファイルの中断
- **WHEN** ヘッダに `CHAR ANSEL` を持つファイルをインポートしようとする
- **THEN** インポートは中断され、「ANSELエンコーディングは未対応」である旨と対処(UTF-8で再エクスポートする等)が表示される

### Requirement: 非対応タグの保全とラウンドトリップ
インポート時、システムが意味を解釈しないタグ・サブツリーを破棄せず内部データモデルに保全しなければならない(SHALL)。同一バージョンで再エクスポートした場合、保全したタグを元の構造で復元しなければならない(SHALL)。異なるバージョンでエクスポートする場合、復元できないタグを警告として明示しなければならない(SHALL)。

#### Scenario: 未知タグの保全と同一バージョンでの復元
- **GIVEN** 独自タグ(例: `_MYTAG`)のサブツリーを含むGEDCOM 5.5.1ファイル
- **WHEN** インポート後、GEDCOM 5.5.1互換モードで再エクスポートする
- **THEN** `_MYTAG` のサブツリーが元の構造のまま出力ファイルに含まれる

#### Scenario: バージョンをまたぐ再エクスポートの警告
- **GIVEN** 5.5.1由来の保全タグを含むデータモデル
- **WHEN** GEDCOM 7.0形式でエクスポートする
- **THEN** 7.0へ変換されない保全タグが警告一覧に列挙される

### Requirement: インポート結果サマリの提示
インポート完了時、システムは取込結果のサマリ(人物数・家族数・判定エンコーディング)と警告一覧を平易な日本語で表示しなければならない(SHALL)。警告は「対象(行番号・タグ)」「内容」「システムの処理(保全した・読み飛ばした等)」を含まなければならない(SHALL)。警告があってもインポート自体は成立しなければならない(SHALL)。

#### Scenario: 警告付きインポートの成立
- **GIVEN** 一部に解釈できない日付値を含むGEDCOMファイル
- **WHEN** インポートする
- **THEN** 解釈できた人物・家族は取り込まれ、サマリに人数と「日付を原文のまま保持した」旨の警告が行番号付きで表示される

#### Scenario: 警告なしインポートのサマリ
- **WHEN** 問題のないGEDCOMファイルをインポートする
- **THEN** 「人物N名・家族M件を取り込んだ」旨のサマリが表示され、警告一覧は空である

### Requirement: ファイル入出力UI
システムはファイル選択とドラッグ&ドロップの両方でインポートを開始できなければならない(SHALL)。エクスポートは形式選択(GEDCOM 7.0 / GEDCOM 5.5.1互換 / JSON)と各形式の用途説明を提示し、ダウンロードとしてファイルを保存しなければならない(SHALL)。ファイルサイズ上限(20MB)を超えるファイルは読み込み前に明示的なエラーとしなければならない(SHALL)。エクスポート時、ファイルに親族の個人情報が含まれる旨の注意書きを表示しなければならない(SHALL)。

#### Scenario: ドラッグ&ドロップでのインポート
- **WHEN** GEDCOMファイルをインポート領域にドラッグ&ドロップする
- **THEN** ファイル選択と同じインポート処理が開始される

#### Scenario: サイズ上限超過の拒否
- **WHEN** 20MBを超えるファイルをインポートしようとする
- **THEN** 読み込みを開始せず、サイズ上限と対処方法が表示される

#### Scenario: エクスポート形式の選択と注意書き
- **WHEN** エクスポートを開始する
- **THEN** 3形式が用途説明付きで提示され、「エクスポートファイルには親族の個人情報が含まれるため取り扱いに注意する」旨が表示される
