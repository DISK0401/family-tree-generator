# Proposal: add-gedcom-json-import-export

## Why

無料版のコア価値である「クライアント完結・データはユーザーの手元」を成立させるには、家系図データをファイルとして持ち出し・持ち込みできる仕組みが不可欠である(サービス終了時のデータ消失不安・ベンダーロックインは競合への主要な不満点のひとつ)。現状リポジトリはプロジェクト雛形のみで家系図データを扱うコードが存在しないため、内部データモデルの定義と、GEDCOM(7.0/5.5.1互換)・JSONのインポート/エクスポートを最初の機能として実装し、以降の編集UI・描画・IndexedDB永続化の土台とする。

## What Changes

- **内部データモデルの新規定義**: 人物・家族関係(婚姻/事実婚/離婚・再婚、養子縁組、複数配偶者)を表現できるTypeScriptのオブジェクト構造を定義する。日付は和暦⇄西暦変換と不完全な日付(年のみ・「頃」等)を扱える形式、氏名は旧字体・異体字・読み仮名を保持できる形式とする。
- **GEDCOMエクスポート**: 内部データモデル → GEDCOM 7.0 ファイル出力。互換性モードとして GEDCOM 5.5.1 形式での出力も選択可能にする。
- **GEDCOMインポート**: GEDCOM 7.0 / 5.5.1 ファイル → 内部データモデルへの変換。文字コード(UTF-8 必須、5.5.1系で流通する他エンコーディングは design で対応範囲を決定)とバージョン自動判定に対応する。解釈できない行・非対応タグは破棄せず警告付きで保全し、インポート結果のサマリ(取込人数・警告一覧)をユーザーへ提示する。
- **JSONエクスポート/インポート**: 内部データモデルをロスレスに保存・復元できるアプリネイティブのJSONスキーマ(スキーマバージョン番号付き)を定義し、入出力を実装する。GEDCOMで表現しきれない情報の完全バックアップ手段と位置づける。
- **ファイル入出力UI(最小限)**: ファイル選択/ドラッグ&ドロップでのインポート、ダウンロードによるエクスポートの最小限のUIを追加する。本格的な編集UI・家系図描画は本変更のスコープ外。
- **README.md 更新**: 対応フォーマット(GEDCOM 7.0 / 5.5.1互換 / JSON)と制限事項を記載する。

破壊的変更: なし(新規機能のみ。既存のプレースホルダーページに機能を追加する)。

### スコープ外(後続チェンジで扱う)

- IndexedDBへの永続化、家系図の描画(family-chart)・編集UI、PNG/PDF出力
- 有償版(クラウド保存・戸籍OCR)との連携

### 無料版 / 有償版への影響

- 対象は**無料版**(クライアント完結)。ただし内部データモデルとGEDCOM/JSON変換ロジックは、有償版の戸籍OCR結果の構造化出力やクラウド保存でも共通利用する前提で設計する。

### 競合調査とUX方針

- MyHeritage・FamilySearch・Gramps は GEDCOM 対応済みで、データ移行手段の提供は業界標準。一方、国内競合(みんなの家系図、買い切りWindowsソフト群)はGEDCOM対応が弱く、移行不安が不満点として顕在化している。GEDCOM入出力は国内市場での明確な差別化ポイント。
- UXで既存製品を上回るための方針: ①インポート時に警告・非対応項目を隠さずサマリ表示(Gramps等は玄人向けでエラーが難解)、②非対応タグもデータとして保全し再エクスポート時に欠落させない、③日本語環境で流通するファイルの文字化けを防ぐエンコーディング処理、④ドラッグ&ドロップ対応の簡単な操作。

### プライバシー・法務への影響

- インポート/エクスポートはすべて**ブラウザ内(クライアントサイド)で完結**し、家系図データ・個人情報をサーバへ送信しない。無料版の絶対制約に適合し、データフローの変更はない(静的アセット配信のみ)。
- エクスポートファイルはユーザー自身の端末に保存されるため、取り扱い注意(親族の個人情報を含む旨)をUI上で注記することを design で検討する。
- 法務上の新たな論点なし(事実証明文書の生成には該当しない)。

### ロールバック方針

- すべて git 管理のフロントエンドコードであり、`git revert` と再デプロイで巻き戻し可能。サーバ側リソース・外部サービス設定の変更はない。
- ユーザーデータ移行の考慮も不要(本変更時点では永続化ストアが存在しないため)。JSONスキーマにはバージョン番号を持たせ、将来の互換性問題に備える。

## Capabilities

### New Capabilities

- `family-data-model`: 家系図の内部データモデル。人物・家族関係(養子縁組・再婚・複数配偶者・事実婚)、不完全日付と和暦対応の日付表現、旧字体・読み仮名を含む氏名表現を定義する。
- `gedcom-import-export`: GEDCOM 7.0 のインポート/エクスポートと GEDCOM 5.5.1 互換モード。バージョン・文字コードの取り扱い、非対応タグの保全、インポート結果サマリの提示を含む。
- `json-import-export`: アプリネイティブJSON形式(スキーマバージョン付き)によるロスレスなエクスポート/インポート。

### Modified Capabilities

なし(既存 spec は `project-scaffold` / `cloudflare-deploy` のみで、要件変更はない)。

## Impact

- **新規コード**: `src/` 配下にデータモデル型定義、GEDCOMパーサ/シリアライザ、JSONシリアライザ、ファイル入出力UIコンポーネント、および各ユニットテスト(Vitest)。
- **更新**: `src/App.tsx`(入出力UIの組み込み)、`README.md`。
- **依存関係**: GEDCOM処理ライブラリ(npmのTypeScript製ツールキット)を採用するか自前実装するかは design で決定。ライセンスはMIT等の許諾的ライセンスに限定する。
- **CI/デプロイ**: 既存のPR品質ゲート・デプロイフローをそのまま利用(変更なし)。
